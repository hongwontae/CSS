1. generic-family와 font-family
제네릭 패밀리가 폰트 패밀리를 포함하고 있다. (제네릭 패밀리가 숲, 폰트 패밀리가 나무)
특정 핵심 속성을 지정하는 제네릭 패밀리 안에서 일반적 속성은 같지만 생김새는 다른
폰트 패밀리가 존재한다.


2. 브라우저 설정
css에서 font가 적용되지 않았다면 => 브라우저의 표준 글꼴이 선택된다.
css에서 글꼴이 적용되면 generic-family, font-family가 적용된다.

font-family을 적용하는 방법은 세 가지가 존재한다.
user의 컴퓨터에 폰트를 설치하는 방법 => 비추
web-fonts 제 3자로 부터 가져와서 적용하는 방법
서버로부터 가져오는 방법


3. 기본 폰트 패밀리 활용하기
font-family을 설정하지 않으면 브라우저가 설정한 기본 폰트을 글꼴로 설정한다.
=> 개발자가 제어할 권한이 없다. 글꼴의 모양과 크기에 의해 요소가 망가질 수 도 있다.


4. font-family 구문 이해하기
font-family : "폰트 패밀리1", "폰트 패밀리2", "제네릭 패밀리"
=> 폰트 패밀리1이 적용되지 않으면 폰트 패밀리2 다음 폰3.... 최후에는 제네릭 패밀리 중에 하나인데 
이는 브라우저가 선택


5. 로컬 저장된 폰트 활용하기
브라우저마다 가지고 있는 폰트는 다르다.
기기에 따라 가지고 있는 폰트 또한 다르다.
=> 사용자가 폰트를 기본적으로 가지고 있거나 브라우저가 지원하지 않으면 글꼴을 통제하기 어렵다.


6. 구글 폰트로 작업하기
폰트를 가져올 떄 타 웹 사이트를 통해 가져오면 적용할 수 있는 방법은 두 가지이다.
=> html에 link를 통해 가져오는 방법
= css 파일에 @import url()를 통해 css 파일에 가져오는 방법

import나 link로 가져왔을 떄 font-family에 value
=>css2?family=...
=> ...을 font-failmy에 value로 주면 된다.


7. font-face와 font-style
웹을 통해 폰트를 가져오면 전체 형식을 쓸 수 있게 가져오는 것이 아닌 400 regular로 가져온다.
(font-weight을 하면 적용되기는 하지만 브라우저가 맞춰주는 것이다. 해당 글꼴에 대한 스타일에 대한 폰트나
스타일이 아니다. import나 link에 추가적으로 가져와야 한다.)
400 => normal/ 700 => bold


8. 사용자 지정 폰트 가져오기 
폰트 다운로드 => 4개의 ttf가 생성
=> @font-face{
    font-family : "사용자 지정 폰트 패밀리";
    src : url("파일주소/파일명");
    font-weight : nnn
} 
@font-face는 파일을 통해 글꼴을 표현하는 방식이다.
@font-face는 인위적으로 글꼴 크기나 스타일을 바꿀 수 없다.
=> 가지고 오는 글꼴이 굵은 파일을 가져와서 font-weight을 지정해줘야 먹힌다.


9. 폰트 포맷 이해하기
가지고 온 글꼴의 파일이 무엇인지 알리는 프로퍼티 => 안 써도 된다.
ttf가 제일 유명하고 브라우저 호환성이 제일 좋다.
format("truetype") => .ttf
format("woff") => .woff
format("woff2") => .woff2
